# Release notes v7.4

## Breaking change: `locationsStrategies` is now abstract

`locationsStrategies` is now an abstract property in the Material rules. Every game must explicitly implement it.

This prevents silent bugs caused by misspelling the property name (which would result in no location strategy being applied).

## New animations API

The `Locator` property `hide` has been **renamed to `ignore`** (completely removes the item from the DOM, previous behavior of `hide`).

The new `hide` now applies `scale(0)` instead of removing the item from the DOM, allowing smooth exit animations.

Animations are now processed at the `DynamicItemsDisplay` level for better performances.

## Disable zoom-pan-pinch

A new option allows you to disable the zoom-pan-pinch feature entirely on the game table.

## Long click: multiple moves

`ItemLocator.itemExtraCss` and other item interactions now support triggering **multiple moves on a long click**.

## `HeaderText` defaults

You can now pass default values to `HeaderText`. This is important for the new translation process, where default values are used as the base text for translations.

## Hide child items automatically

When a parent item is hidden, its child items are now hidden by default. This removes the need to manually handle child visibility.

## i18n improvements

- The board game name (i18n) is now used as the document title
- New `VITE_I18N_DEBUG` environment variable to enable i18next debug mode

## Performance improvements

- Prevented useless rerenders in `StaticItems`
- Fixed `FocusProvider` and `NoZoomFocusProvider` to avoid unnecessary `ItemDisplay` rerenders
- Moved `useUndo` hook outside `DraggableMaterial` to reduce rerenders
- Prevented `DraggableMaterial` from rerendering on every game state change

## Bugfixes

- Fixed undo: going back to a simultaneous action phase is no longer possible
- `game.view` is now preserved on undo
- Fixed item creation during simultaneous phases (duplicate index issue)
- Fixed controls on move/delete for items with no quantity
- Fixed generics typing in material moves typeguards
- Fixed `HeaderText` creating useless translation keys when there are no active players
- Fixed `usePlayerName` calling `optionsSpec` with `undefined`
- Fixed `react-i18next` warning on first render
- Fixed chat translation key
- Removed circular dependency between `HiddenMaterialRules` and `SecretMaterialRules`

## Other

- Switched to **vitest** for unit tests
- `@gamepark/api-types` is now a peer dependency in `react-client`
