# Release notes v7.7

## Breaking change: translations are now stored in git

Translations are no longer loaded from a centralized S3 bucket (`translations.game-park.com`). Each game must now host its own translation files in the repository.

### What changed

- **Game translations** must be placed in `app/public/translation/{locale}.json` (e.g. `en.json`, `fr.json`)
- The game namespace is loaded from the game's own origin (`/translation/{locale}.json`)
- **Common translations** (`common`, `credits`) are now loaded from the main Game Park website instead of S3

### Migration steps

1. **Download your game's translations** from S3: `https://translations.game-park.com/{locale}/{game}.json` (e.g. `https://translations.game-park.com/en/mythologies.json`)
2. Create the directory `app/public/translation/`
3. Save each locale file as `en.json`, `fr.json`, etc.
4. Each file is a flat key-value object with your game's translation keys:
```json
{
  "card.1": "Knight",
  "card.2": "Archer",
  "header.play": "{player} plays a card"
}
```
5. **Redeploy** your game — the translation files are deployed automatically with `yarn deploy` (rclone syncs the `public/` directory)

### Missing keys

Missing translation keys are now logged as warnings (dev) or errors (production) in the console, and captured by Datadog. The old `/api/translations/missing` endpoint still exists for backward compatibility but only logs errors — it no longer stores keys in a database.

Missing keys should be treated as bugs to fix in the code: always provide at least the English translation.
