# Release notes v7.5

## Theme customization

Games can now fully customize the look and feel of all framework-provided UI (menus, dialogs, headers, player panels, journal, tutorials…) through a new `theme` prop on `GameProvider`.

### Color palette

A new `PaletteTheme` system defines 13 semantic color tokens (primary, surface, danger, disabled) with their interactive states. Setting `theme.palette.primary` to a different color automatically updates all buttons, borders and accents across the UI.

A default palette preserving the existing teal brand is provided — existing games see zero visual changes.

### Per-component CSS overrides

The `GameTheme` interface exposes optional sub-theme interfaces for fine-grained styling:

- `DialogTheme`: container, backdrop, close icon, content
- `MenuTheme`: panel, buttons, main button, pop button
- `JournalTheme`: tab, selected tab, chat bar, history entry
- `ResultTheme`: border, icon, container, close icon
- `HeaderTheme`: bar, buttons
- `PlayerPanelTheme`: active ring colors, panel, data badge
- `TutorialTheme`: container, content
- `TimeStatsTheme`: container, think/wait backgrounds

### CSS custom properties

Theme colors are now exposed as CSS custom properties (prefixed `--gp-`), removing the need for `useTheme()` in most components. A new `--gp-scale` variable provides unified responsive sizing (replacing scattered media queries).

## Customizable Material help display

The `MaterialDescription` class now exposes a `helpDisplay` property that lets you fully replace the item rendering area of the material help dialog.

The default rendering has been extracted into a `DefaultHelpDisplay` component, so you can either:
- **Override entirely**: set `helpDisplay` to your own component for custom item positioning, overlays, or alternative views.
- **Wrap the default**: import `DefaultHelpDisplay` and compose around it.

The custom component receives the `item`, `itemType`, `itemIndex`, `displayIndex` and a `closeDialog` callback.

## `GridLocator` for rectangular grid positioning

A new `GridLocator` places items on a rectangular grid using `location.x` and `location.y` as cell coordinates, with configurable gaps between cells.

### Automatic panning

When items grow beyond a defined visible window (`gridSize`), the grid automatically pans to keep them in view:
- Without `gridSize`: the view always centers on the barycentre of all items.
- With `gridSize` and items within bounds: no movement occurs.
- With `gridSize` and overflow: only the overflowing axis recenters.

The panning offset is recomputed once per grid per game state change for optimal performance.

### Usage

```typescript
const gridLocator = new GridLocator({
  gap: { x: 5, y: 5 },
  gridSize: { columns: 7, rows: 5 },
})
```

Multiple independent grids are supported through `getGridId()` (same pattern as `PileLocator.getPileId`).

## `changeView` in `MaterialMoveBuilder`

`changeView` is now available in `MaterialMoveBuilder`, allowing view changes to be composed with other material moves.

## `Material.exists`

A new `Material.exists` method is available in the rules API, providing a convenient way to check whether any material item matches a given filter.

## Sequential animation for rules moves

A new API prevents certain moves from being animated in parallel with moves happening before or after them.

On the **rules-api** side, you can now mark rules moves as sequential, ensuring they animate one after another rather than simultaneously. The interleaving logic now uses `item.location.player` (the owner of the item) instead of `context.player` (the player that triggered the action) for more accurate animation ordering.

On the **react-client** side, rules transition moves (such as `StartSimultaneousRule`) are no longer animated in parallel with other actions. The animation lifecycle and legal moves lifecycle have also been improved overall.

## DevToolsHub dev panel

A new **DevToolsHub** developer panel has been added to `react-game` (since 7.5.4), with support for game options. This panel provides a centralized place for development tools during game testing.

## Bugfixes

- Fixed interleaving for item creation during simultaneous game phases
- Improved recovery from mismatch consequences errors
