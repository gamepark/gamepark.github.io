# Migration to framework v7.7

## Translations are now stored in git

Translations are no longer loaded from the S3 bucket (`translations.game-park.com`). Each game must now host its own translation files in the repository.

### Migration steps

1. **Download your game's translations** from S3: `https://translations.game-park.com/{locale}/{game}.json` (e.g. `https://translations.game-park.com/en/mythologies.json`)
2. Create the directory `app/public/translation/`
3. Save each locale file as `en.json`, `fr.json`, etc.
4. Each file is a flat key-value object with your game's translation keys:
```json
{
  "card.1": "Knight",
  "card.2": "Archer",
  "header.play": "{player} plays a card"
}
```
5. **Redeploy** your game — the translation files are deployed automatically with `yarn deploy` (rclone syncs the `public/` directory)

:warning: Do not include common translations (`common`, `credits`) in your files — they are now loaded from the main Game Park website.

### Missing keys

Missing translation keys are logged as warnings (dev) or errors (production) in the console, and captured by Datadog.

:bulb: During development, you can write translations only in your native language. Translations to other languages will be done later. Add `?locale=fr` (or `es`, `en`...) to the URL to test the texts in your language.
